# LangChain Agent 开发指南（示例知识库文档）

## Agent 基本概念

Agent 是 LLM 应用的核心组件，它使用 LLM 作为推理引擎，决定要执行哪些操作以及执行顺序。

### ReAct 模式

ReAct（Reasoning + Acting）是 Agent 最经典的模式：
1. Thought：LLM 分析当前状态，决定下一步
2. Action：调用工具获取信息
3. Observation：解读工具的返回结果
4. 重复直到得出最终答案

### Tool 设计原则

- 每个工具职责单一
- 描述清晰，让 LLM 理解何时该调用
- 返回结果结构化，便于 LLM 解读
- 做好异常处理，失败时不崩溃

### Tool 注册机制

使用 LangChain 的 @tool 装饰器 + importlib 自动扫描，新增工具只需：
1. 在 tools/ 目录下创建 .py 文件
2. 用 @tool 装饰器标记函数
3. 重启应用即可自动发现

### 记忆管理

Agent 的记忆分为短期和长期：
- 短期记忆：当前会话的对话历史（内存）
- 长期记忆：持久化的用户画像和偏好（SQLite + Chroma）

### RAG（检索增强生成）

RAG 让 LLM 在回答前先检索相关文档，提高准确性：
- 文档加载 → 文本切分 → 向量化 → 存入 Chroma
- 查询扩展 → 混合检索（向量 + BM25） → LLM 加工

### Hybrid RAG

混合检索 = 向量语义检索 + BM25 关键词检索：
- 向量检索：擅长语义相似（"酒店" ≈ "住宿"）
- BM25：擅长精确匹配（"希尔顿" → 命中"希尔顿酒店"）
- 两者去重合并，弥补各自的盲区
